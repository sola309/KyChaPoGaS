# KyChaPoGaS クリエイティブ・ロードマップ

> 目標: 「素材の切り貼り」を超えた、**新しい映像表現を発明できる**MAD制作ツール。
> 音MAD・静止画MAD・ゲームOP風など、あらゆるMAD表現手法を取り込み拡張する。

## 実装済みの表現基盤

| 機能 | 説明 |
|---|---|
| ビート解析 + 音ハメスコア | BPM/ビート検出、ビートと視覚変化の一致を採点、弱ビートを提示 |
| フレーム差分カーブ | 全フレームの画面変化量を数値化。クリップ上に波形表示（音声波形の動画版） |
| リズムレーン | タイムライン全体の「変化量×ビート」を一望。琥珀▼=画が止まっている拍 |
| トランジション | カット/クロス/白フラッシュ/黒。尺保存型(音ズレなし) |
| 速度カーブ | プリセット+任意ベジェ(グラフエディタ)。区分一定近似でレンダリング |
| コードMG | HTML/CSS/JSをフレーム正確に動画化。テンプレ+カスタム、**透過背景対応** |
| マルチトラック合成 | 2本目以降のVideoトラックをオーバーレイ（不透明度・スクリーン/加算/乗算） |
| AI編集 | LLM/MCPツール群で解析→提案→編集→検証のループ |

## 次の表現アイデア（優先度順）

### 1. トランスフォームキーフレーム（静止画MADの核）
クリップに position/scale/rotation のキーフレームを持たせ、Ken Burns・
ズームパンチイン・画面シェイクをタイムラインで打てるように。
ffmpeg `zoompan`/`rotate`/`crop` で実装可能。静止画MADはこれが本体。

### 2. ビート同期エフェクトの自動散布
「全ダウンビートに2フレームの白フラッシュ」「サビ区間の拍ごとにパンチイン」を
一括適用するコマンド。スコアラーの弱ビート出力と組み合わせて半自動音ハメ。

### 3. 音MAD基盤
音声クリップのピッチシフト/タイムストレッチ(rubberband)、サンプル単位の
切り刻み配置、波形からのワンショット検出。

### 4. コードMGの拡張
- **データ駆動MG**: ビート/歌詞/モーションカーブをMGテンプレートに注入
  （`window.seek(t)` に加え `window.data = {beats, lyrics, motion}`）。
  → 音に反応するビジュアライザやリリックモーションが書ける
- Three.js/WebGL テンプレート（3Dカメラワーク、パーティクル）
- ゲームOP風テンプレ集（タイトル・キャラ紹介カットイン・スペック表示帯）

### 5. シーンチェンジ表現の拡充
xfadeの全トランジション(wipe/slide/circleopen/pixelize等)の開放、
マスクワイプ（任意形状）、ディスプレイスメント遷移。

### 6. プレビューの合成対応
現在オーバーレイは書き出しのみ反映。プレビューでの簡易合成
（CSS mix-blend-mode で近似）を検討。

## 設計メモ

- オーバーレイは「基底トラックに1パスずつ合成」方式。クリップ数が増えたら
  filter_complex 一括グラフ化を検討
- 透過は qtrle .mov（互換性優先）。サイズが問題になれば ProRes 4444 / VP9 alpha
- コードMGは `document.getAnimations()` への currentTime 直接シークで決定論的
  （CSS/WAAPI）。JSは `window.seek(t_ms)` 規約
